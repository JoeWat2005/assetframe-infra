# Storage — Cloudflare R2

## Bucket

A single **private** bucket, `assetframe-pro` (the value in `.env.example`'s `R2_BUCKET`). "Private" means there is no public-read policy: objects are only retrievable with a credentialed/presigned request. Both free **and** Pro report files live here — privacy is enforced by the gated route, not by the bucket being Pro-only despite the name.

## Client construction

```ts
const client = (accountId && accessKeyId && secretAccessKey)
  ? new S3Client({
      region: "auto",
      endpoint: `https://${accountId}.r2.cloudflarestorage.com`,
      credentials: { accessKeyId, secretAccessKey },
    })
  : null;
export const r2Configured = Boolean(client && bucket);
```

- R2 is S3-compatible, so the AWS S3 SDK is used with `region:"auto"` and the R2 endpoint.
- The client is `null` unless **all three** credential vars are present. Callers must handle `null`.
- `import "server-only"` — credentials can never reach the client bundle.

## Functions

### `signedReportUrl(key, expiresIn = 120): Promise<string | null>`
Returns a presigned `GetObject` URL valid for `expiresIn` seconds. `null` if R2 isn't configured (so the route can 503 cleanly). The default is **120s**; the gated route passes 600s for public previews and 120s for gated files. See [signed-urls.md](./signed-urls.md).

### `getObjectText(key): Promise<string | null>`
Reads an object's bytes as a string (`Body.transformToString()`). Used by the MCP/API tools to return report **content** (HTML/text) to an agent rather than redirecting a browser. Returns `null` if R2 is unconfigured or the object is missing (try/catch swallows errors).

### `r2Configured: boolean`
Cheap configured-check used by routes/tools to decide whether to attempt R2 at all.

## Credentials & scope

- Credentials are R2 S3 API tokens (`R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY`) scoped to the account/bucket.
- They live only in server env (Vercel server env + local `.env.local`); they are never inlined into the client (no `NEXT_PUBLIC_` prefix).
- The only operations the code performs are `GetObject` (read) and presign. There is **no** put/delete in `lib/r2.ts`.

## CSP interaction

Signed R2 URLs resolve to `*.r2.cloudflarestorage.com`. The CSP `img-src` allow-lists `https://*.r2.cloudflarestorage.com` so the preview thumbnails render. (HTML/PDF report files are delivered by a top-level redirect/navigation to the signed URL, not embedded, so they aren't constrained by `img-src`.) See [../security/csp.md](../security/csp.md).

## Failure modes

| Condition | Result |
| --- | --- |
| Any R2 env var missing | `client = null`, `signedReportUrl` → `null`, route → **503** |
| Object missing | presign still returns a URL; the eventual R2 fetch 404s. `getObjectText` returns `null` |
| Network/SDK error in `getObjectText` | caught → `null` |
