# Cloudflare R2

All report files — free Snapshots, Pro reports, and preview thumbnails — live in a **private** Cloudflare R2 bucket. Nothing is served from the public Next.js bundle; every file is fetched through a gated handler that issues a short-lived signed URL.

## Bucket

- **Name:** `assetframe-pro` (`R2_BUCKET`).
- **Access:** private. The bucket is reached over R2's S3-compatible API at `https://<R2_ACCOUNT_ID>.r2.cloudflarestorage.com` with region `auto` (`lib/r2.ts`).
- **Credentials:** `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET` (see `environment-variables.md`). They are used server-side by the app and by `publish.py` (which auto-loads them from `web/.env.local`). Credentials never reach the browser.

## Object key layout

Keys follow the strict grammar enforced by `lib/report-key.ts` (`<date>/<slug>/<file>`):

```
<YYYY-MM-DD>/<SLUG>/free.html
<YYYY-MM-DD>/<SLUG>/free.pdf
<YYYY-MM-DD>/<SLUG>/pro.html
<YYYY-MM-DD>/<SLUG>/pro.pdf
<YYYY-MM-DD>/<SLUG>/preview.png
```

`<SLUG>` is `[A-Za-z0-9_-]+` (e.g. `BTC`, `AAPL`, `SOL_2`, `BRK-B`). Any other key shape is rejected before R2 is contacted (see `../testing/security-tests.md`).

## Access tiers (who can fetch what)

Served only through `app/api/report/[...key]/route.ts`, which classifies the key, checks entitlement server-side, then 302-redirects to a presigned URL:

| File | Tier | Gate | Signed URL TTL | Cache |
| --- | --- | --- | --- | --- |
| `preview.png` | public | none (marketing thumbnail) | 600s | `public, max-age=300, s-maxage=300` |
| `free.*` | free | signed-in Clerk account | 120s | `private, no-store` |
| `pro.*` | pro | active subscription (`ent.subscribed`) | 120s | `private, no-store` |

- A signed-out user hitting a free/pro file is redirected to `/sign-in` with `redirect_url` preserved; a free user hitting a pro file is redirected to `/pricing`.
- Pro fetches are logged to `download_log` (deduped per user/report/kind/hour) — best-effort, never blocks the download.
- If R2 is unconfigured, `signedReportUrl` returns null and the route returns **503** cleanly.

The MCP/API layer reads object **text** (not a redirect) via `getObjectText` for `get_report`/`get_pro_report`, subject to the same auth.

## Uploading (publish)

`python scripts/publish.py` uploads the free + Pro + preview files for an edition to R2 under the keys above. It auto-loads the `R2_*` vars from `web/.env.local`, and supports `--dry-run` and `--date YYYY-MM-DD`. See `../operations/publication-workflow.md`.

## Setup

1. Create the `assetframe-pro` bucket in Cloudflare R2 (private; no public bucket access, no public dev URL needed).
2. Create an R2 API token (S3 credentials) scoped to that bucket; capture the account id, access key, secret.
3. Put them in `web/.env.local` and in Vercel (Production + Preview).
4. CORS: not required for the download path (the app redirects to the signed URL; the browser then does a normal top-level fetch). `*.r2.cloudflarestorage.com` is allow-listed in the CSP `img-src` for the public preview thumbnails (`next.config.ts`).

## Security notes

- The signed URL is the only thing the client ever sees; it expires in 120s for gated files, 600s for public previews.
- Object-key validation is an anchored allow-list that blocks traversal/encoding bypasses (`../testing/security-tests.md`).
- Unpublishing an edition (admin "Hide") flips the DB `hidden` flag — **the R2 files are untouched** and the edition can be restored. Files are not deleted by hiding (`../admin/maintenance.md`).
